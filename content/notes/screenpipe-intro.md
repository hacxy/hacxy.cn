你有没有这种经历：昨天在电脑上看到一个很重要的东西，今天怎么也找不到了？或者开完会想回顾某个细节，却发现记忆已经模糊？

[Screenpipe](https://github.com/screenpipe/screenpipe) 就是为了解决这个问题而生的。它能 24/7 记录你的屏幕和麦克风音频，然后让 AI 帮你搜索和回忆任何你经历过的事情。关键是——所有数据都存在本地，不会上传到任何云端。

## 这是什么？

Screenpipe 是一个开源项目，由 Y Combinator 孵化（S26 批次），目前在 GitHub 上有 18.9k star。它的核心理念很简单：

```
屏幕 + 音频 → 本地存储 → AI
```

把你电脑上发生的一切都记录下来，然后用 AI 让这些数据变得可搜索、可利用。

## 为什么需要它？

现在 AI 助手最大的问题是什么？是缺乏上下文。你跟 ChatGPT 说"帮我总结一下昨天的会议"，它根本不知道你昨天开了什么会。

Screenpipe 就是来解决这个问题的。它给 AI 提供了"记忆"——你过去几天在电脑上做了什么、看了什么、说了什么，AI 全都知道。

对于开发者来说，这个价值更明显。你可以让 Cursor、Claude Code 这些 AI 编程助手知道你最近在写什么代码、查什么文档、调试什么问题，这样 AI 的建议会精准很多。

## 核心功能

### 事件驱动的屏幕录制

Screenpipe 不是傻傻地每秒都截图。它会监听系统事件——应用切换、点击、打字暂停、滚动——只在真正有变化的时候才截图。每张截图都会配上系统的可访问性树（Accessibility Tree），也就是操作系统已经知道的结构化文本：按钮、标签、文本框。

如果可访问性数据不可用（比如远程桌面、游戏），它会退回到 OCR 识别。这样既能保证数据质量，又能最大程度节省 CPU 和存储空间。

### 音频转录

它能同时录制系统音频（你听到的）和麦克风输入（你说的）。使用 OpenAI Whisper 在本地进行实时语音转文本，支持说话人识别和分离。Zoom、Google Meet、Teams 等任何音频源都能处理。

### AI 搜索

这是最实用的功能。你可以用自然语言搜索所有记录的内容——屏幕文字、OCR 识别的文本、音频转录。还能按应用名、窗口标题、浏览器 URL、日期范围进行过滤。

比如你可以说"上周三在 Slack 里讨论的那个 API 设计方案"，Screenpipe 就能帮你找到。

### 插件系统（Pipes）

Pipes 是 Screenpipe 的插件系统，本质上是用 Markdown 文件定义的定时 AI 任务。每个 Pipe 就是一个 `pipe.md` 文件，里面有提示词和执行计划。

内置的 Pipes 包括：
- **Obsidian 同步**：自动把屏幕活动同步到 Obsidian 笔记
- **提醒事项**：扫描活动中的待办事项，创建 Apple Reminders
- **创意追踪器**：从浏览内容中发现创业灵感

开发者可以在 `~/.screenpipe/pipes/` 目录下创建自己的 Pipes。

### MCP 服务器

Screenpipe 可以作为 MCP（Model Context Protocol）服务器运行，让 AI 助手直接查询你的屏幕历史。支持 Claude Desktop、Cursor、VS Code 等任何 MCP 兼容的客户端。

配置超简单：

```bash
claude mcp add screenpipe -- npx -y screenpipe-mcp
```

然后你就可以问 Claude"我过去 5 分钟看到了什么？"或者"总结一下今天的对话"。

## 隐私和安全

这是 Screenpipe 最让我放心的地方：

- **100% 本地存储**：所有数据都存在你设备的本地 SQLite 数据库中，不会发送到任何外部服务器
- **开源透明**：MIT 协议，代码完全可审计
- **支持本地 AI**：可以用 Ollama 或任何本地模型，数据完全不出本机
- **无需注册**：核心功能不需要任何账号
- **数据归你所有**：随时可以导出、删除或备份
- **可选加密同步**：设备间同步使用端到端加密（零知识加密）

## 技术架构

Screenpipe 用 Rust 编写，性能优秀：

- **CPU 占用**：5-10%（得益于事件驱动的录制方式）
- **内存占用**：0.5-3GB
- **存储空间**：约 20GB/月（8 小时约 300MB，比连续录制的 2GB 少得多）

技术栈：
- 桌面应用：Tauri（Rust + TypeScript）
- 数据库：SQLite + FTS5 全文搜索
- API：REST API 运行在 localhost:3030
- 语音识别：OpenAI Whisper（本地）或 Deepgram（云端）

## 我的使用体验

说实话，刚开始用的时候我有点担心隐私问题——毕竟 24/7 录制屏幕听起来很吓人。但用了几天之后就放心了，因为：

1. 数据真的只在本地，网络监控工具可以验证
2. 可以设置过滤规则，排除敏感应用（比如密码管理器）
3. 开源代码完全透明，社区也在持续审计

最让我惊喜的是搜索功能。上周我跟同事在 Slack 讨论了一个技术方案，但没保存。三天后需要回顾的时候，直接用自然语言搜索就找到了，连当时的截图都有。

对于开发者来说，配合 Claude Code 使用效果拔群。AI 能看到你最近在调试什么、查什么文档，给出的建议会精准很多。

Screenpipe 解决了 AI 的一个根本问题：缺乏个人上下文。它让 AI 能够"记住"你经历过的一切，同时又通过本地存储保护了你的隐私。

如果你是那种经常需要回忆"昨天在电脑上做了什么"的人，或者想让 AI 助手更懂你，Screenpipe 值得一试。
