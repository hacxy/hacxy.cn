# 第一章：Claude Code 是什么

如果你用过 ChatGPT 或者其他 AI 聊天工具来写代码，你一定遇到过这种情况：把一段代码粘进去，让它帮你修，它给出了答案，但你不确定这个修改放到整个项目里是否正确；或者你描述了一个需求，它生成了代码，但和你的已有架构完全对不上。

Claude Code 解决的正是这个问题。

## 一句话描述

Claude Code 是 Anthropic 官方开发的 AI 编程助手，以命令行工具（CLI）的形式运行在你的终端里。它能直接读取你的项目文件、执行命令、修改代码，并在真实的开发环境中完成任务——不是生成一段"示例代码"让你自己去套，而是直接在你的项目里做事。

## 和普通 AI 聊天有什么不同

普通 AI 聊天工具（包括网页版 Claude）的工作方式是"你说我答"：你把代码贴给它，它输出修改后的代码，但它看不到你的完整项目，不知道其他文件里有什么，也无法直接运行或修改文件。

Claude Code 的工作方式完全不同：

| 对比维度 | 普通 AI 聊天 | Claude Code |
|---------|------------|-------------|
| 代码访问 | 你粘贴什么它就看什么 | 直接读取整个项目 |
| 文件操作 | 输出建议，你手动修改 | 直接编辑文件 |
| 命令执行 | 无法执行 | 可运行测试、构建等命令 |
| 上下文范围 | 单次对话窗口 | 项目全局 + 持久记忆 |
| 工作方式 | 问答式 | 代理式（主动完成任务） |

这种差异意味着你可以对 Claude Code 说"帮我把登录流程重构成 JWT"，它会自己去找相关文件、理解当前架构、做出修改、运行测试验证，而不是给你一段通用的 JWT 示例代码。

## Claude Code 能做什么

### 理解代码库

你可以直接问它关于项目的问题，它会自己去读相关文件：

- "这个项目的认证逻辑在哪里？"
- "为什么首页加载这么慢？"
- "这个函数被哪些地方调用？"

### 修改代码

告诉它要做什么，它会跨文件完成修改：

- "把所有用 `var` 声明的变量改成 `const` 或 `let`"
- "给 `UserService` 里的每个公共方法加上错误处理"
- "把这个组件拆成三个更小的组件"

### 执行命令

它可以运行你项目里的命令：

- 跑测试并根据失败结果修复代码
- 执行构建，排查报错
- 运行 lint，修复所有 warning

### Git 操作

- 分析 staged 区的变更，生成规范的提交信息
- 创建分支、提交、甚至开 PR
- 解决合并冲突

### 调试

- 从报错信息出发，追踪问题根源
- 分析测试失败的原因，给出修复方案

## 运行在哪里

Claude Code 有几种使用方式：

**终端 CLI**（最核心的方式）：在项目目录下运行 `claude` 命令，进入交互式对话。

**VS Code / JetBrains 插件**：在 IDE 里直接唤起，可以用 `@` 引用当前编辑的文件，看到内联 diff。

**桌面应用**：独立的 macOS / Windows 应用，界面更友好。

**Web 端**：`claude.ai/code`，无需安装，打开即用（功能略有限制）。

## 需要什么账号

使用 Claude Code 需要：

- **Claude 订阅账号**（Pro / Max / Team / Enterprise），或者
- **Anthropic Console 账号**（按用量付费）

它本质上是 Claude 大模型的能力加上编程工具链的结合。模型越强（Opus > Sonnet > Haiku），处理复杂任务的能力越好，但消耗的 token 也更多。

## 小结

Claude Code 不是一个更聪明的代码补全工具，也不是一个会写代码的聊天机器人。它更接近于一个**能独立工作的初级工程师**——你告诉它目标，它自己去探索、去修改、去验证，最后把结果交给你审查。

理解这一点，是用好 Claude Code 的前提。
