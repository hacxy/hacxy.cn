# write-post

这个 skill 帮你在 hacxy.cn 博客中撰写技术文章。

## 项目约定

文章存放在项目根目录的 `posts/` 文件夹，使用 Markdown 格式，支持 YAML frontmatter。

**Frontmatter 结构：**

```yaml
---
title: 文章标题
date: YYYY-MM-DD
tags:
  - 标签1
  - 标签2
summary: 一段简洁的文章摘要，配置后将在文章顶部以模拟 AI 打字输出的形式展示。（可选）
---
```

`summary` 字段为可选。建议在内容较长、知识密度较高的技术文章中加入，帮助读者快速了解文章核心内容。摘要应为 1–3 句话，语言简洁，突出文章的核心结论或要点，避免复述标题。

文件名使用小写英文 + 连字符（slug），例如 `react-hooks-guide.md`。

**常用 tags 参考（根据内容自行选择）：**
- 前端、CSS、JavaScript、TypeScript、React、Vue、Node.js
- 工具链、Vite、构建、工程化
- 随笔、思考、经验总结
- 性能、安全、最佳实践

## 写作风格指南

风格根据文章主题灵活切换：

- **技术教程型**（如"如何实现 X"）：步骤清晰，配代码示例，实操性强，语言简练直接
- **深度分析型**（如"为什么选择 X"、"X vs Y"）：有自己的观点，讲清楚背后的原因，可以有对比表格
- **随笔/思考型**（如经验总结、感悟）：语气自然随意，不需要严格结构，第一人称

**通用原则：**
- 中文为主，技术术语保留英文（如 useState、Promise、TypeScript）
- 开篇直接切入主题，不要废话铺垫
- 代码示例用三反引号 + 语言标识（```typescript）
- 标题层级不超过三级（#、##、###）
- 结尾可以有总结，但不必强制

## 工作流程

### 模式 1：起草完整文章

收到话题或简单描述后，直接生成完整文章，然后写入 `posts/<slug>.md`。

步骤：
1. 根据话题判断合适的风格
2. 选取合适的 tags（2-4 个）
3. 生成文章，包含完整 frontmatter
4. 确认后写入 `posts/<slug>.md`

**示例：**

用户说："写一篇关于 TypeScript 泛型的文章"

生成文件名：`typescript-generics.md`

```markdown
---
title: TypeScript 泛型入门
date: 2025-05-06
tags:
  - TypeScript
  - 前端
---

# TypeScript 泛型入门

泛型是 TypeScript 中最强大的特性之一...
```

### 模式 2：文章大纲规划

如果用户想先规划结构，给出大纲：列出各节标题 + 每节一句话说明要点，等用户确认后再展开写。

### 模式 3：扩展 / 润色现有内容

用户提供草稿片段时：
- **扩展**：补充细节、添加代码示例、丰富说明
- **润色**：保持原意，改善表达流畅性和逻辑性，不要过度改写

### 写入文件

生成文章后，询问用户是否直接写入（或用户一开始就要求写入时直接写）。写入路径：

```
posts/<slug>.md
```

slug 从标题推导：中文标题转为英文描述性短语，英文标题直接转 kebab-case。

## 日期

使用今天的实际日期作为 frontmatter 中的 date（格式 YYYY-MM-DD）。
